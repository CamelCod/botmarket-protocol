# Contributing to BotMarket Protocol

Thank you for being here. This protocol is built by the community, for the community.

## Ways to Contribute

### 1. Propose a Spec Change (RFC Process)
- Open an issue tagged `[RFC]`
- Describe: the problem, your proposed change, and why
- Community discussion: minimum 14 days
- Merge requires 2 maintainer approvals
- Your name goes in the spec changelog

### 2. Build an Implementation
- Build a node, SDK, agent, or tool
- Open a PR to add it to README's ecosystem list
- No approval needed — just build and link it

### 3. Report Issues
- Ambiguities in the spec
- Missing edge cases
- Security concerns (please use private disclosure for security issues)

### 4. Improve Documentation
- Examples, diagrams, translations
- All welcome, no RFC process required

## Code of Conduct
Be kind. Be direct. Be constructive. This is a technical community, not a social network.

## Recognition
Every contributor who gets a PR merged is listed in CONTRIBUTORS.md permanently.
Spec RFC authors are credited in the spec itself.

Your name in an open protocol spec lives longer than any LinkedIn post.
